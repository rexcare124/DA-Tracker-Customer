Common Error Messages and Solutions

❌ Error: "Invalid credentials" or "Authentication failed"

When it occurs: When calling external APIs (LlamaParse, OpenAI)

Solution:
  1. Check that your API key is correct
  2. Verify the API key hasn't expired
  3. Ensure the credential is properly saved in n8n
  4. Test the credential by clicking "Test" button in credential settings
  5. Check API key format (e.g., "Bearer " prefix for some APIs)
💡 Prevention: Always test credentials immediately after creating them. Use the "Test" button in the credential configuration.

❌ Error: "Cannot read property 'X' of undefined"

When it occurs: In Code nodes or expressions when accessing data from previous nodes

Solution:
  1. Check the output of the previous node to see the actual data structure
  2. Use optional chaining: {{ $json?.field?.subfield }}
  3. Add null checks in Code nodes: 
    if ($json && $json.field) {
  // Access $json.field
}
  4. Use the Expression Editor's data preview to see available fields
⚠️ Debug Tip: Click on the previous node and check the "Output" tab to see exactly what data is being passed.

❌ Error: "Request failed with status code 400/401/403/404/500"

When it occurs: HTTP Request nodes failing to connect to APIs

Solution:
  1. 400 Bad Request: Check request body format, required fields, data types
  2. 401 Unauthorized: Verify API credentials and authentication method
  3. 403 Forbidden: Check API key permissions and rate limits
  4. 404 Not Found: Verify the API endpoint URL is correct
  5. 500 Server Error: API service issue - check API status page, retry later

Additional checks:

  • Review the "Response" tab in the HTTP Request node to see error details
  • Check API documentation for correct request format
  • Verify headers are set correctly
  • Test the API endpoint with a tool like Postman or curl

❌ Error: "Workflow execution timed out"

When it occurs: Workflow takes too long to execute

Solution:
  1. Break workflow into smaller chunks
  2. Optimize Code nodes (avoid nested loops on large datasets)
  3. Use "Wait" nodes strategically instead of polling in loops
  4. Check for infinite loops in your code
  5. Increase timeout settings if available (self-hosted n8n)
  6. Process data in batches using "Split In Batches" node

❌ Error: "File not found" or "Permission denied"

When it occurs: Write Binary File node or file system operations

Solution:
  1. Verify the directory path exists
  2. Check file system permissions (read/write access)
  3. Use absolute paths instead of relative paths
  4. For cloud n8n, use cloud storage (Google Drive, S3) instead of local file system
  5. Create the directory structure before running the workflow

❌ Error: "Webhook URL not accessible"

When it occurs: Webhook node not receiving requests

Solution:
  1. Ensure the workflow is activated (Active toggle is ON)
  2. Verify the webhook URL is correct and copied completely
  3. Check if n8n instance is accessible from the internet (for external webhooks)
  4. For self-hosted n8n, configure port forwarding or use ngrok for testing
  5. Check firewall settings
  6. Verify the HTTP method matches (POST, GET, etc.)

Workflow-Specific Issues

❌ LlamaParse API: "Parsing failed" or "Invalid file format"

Solution:
  • Verify the PDF file is not corrupted
  • Check file size limits (LlamaParse has size restrictions)
  • Ensure the file is actually a PDF (not just renamed with .pdf extension)
  • Try a different PDF file to isolate the issue
  • Check LlamaParse API status and documentation for current limits

❌ OpenAI: "Rate limit exceeded" or "Insufficient quota"

Solution:
  • Check your OpenAI account billing and quota
  • Add rate limiting/delays between API calls
  • Use "Wait" node to add delays
  • Consider using gpt-3.5-turbo for cost savings (if acceptable quality)
  • Implement retry logic with exponential backoff

❌ Code Node: "SyntaxError" or "ReferenceError"

Solution:
  1. Check JavaScript syntax (missing brackets, quotes, etc.)
  2. Verify variable names are correct
  3. Use the "Test Step" feature to debug
  4. Check console/execution logs for detailed error messages
  5. Validate JSON parsing with try-catch blocks: 
    try {
  const data = JSON.parse($json.response);
} catch (error) {
  return { json: { error: error.message } };
}

❌ JSON Output: "Invalid JSON structure" or missing fields

Solution:
  • Validate JSON structure before saving
  • Add data validation in Code nodes
  • Check OpenAI response format matches expected schema
  • Use JSON schema validation in OpenAI node
  • Add fallback values for missing fields: 
    const staff = $json.staff || [];
const departments = $json.departments || [];

Debugging Tips and Techniques

🔍 Tip 1: Use Execution History

Always check the "Executions" tab to see what data passed through each node. This helps identify where the workflow failed.

🔍 Tip 2: Test Individual Nodes

Right-click any node and select "Execute Node" to test it in isolation. This helps identify which node is causing issues.

🔍 Tip 3: Add Debug Nodes

Add temporary Code nodes that log data: 

// Debug node
console.log('Debug data:', JSON.stringify($json, null, 2));
return $input.all();

🔍 Tip 4: Use Expression Editor Preview

When writing expressions, use the preview feature to see what data is available and test your expressions before saving.

🔍 Tip 5: Check Node Output

Click on each node after execution and review the "Output" tab. Compare expected vs actual data structure.

🔍 Tip 6: Enable Verbose Logging

For self-hosted n8n, enable verbose logging in settings to see detailed execution information.

Performance Issues

🐌 Workflow runs very slowly

Optimization Strategies:
  • Batch Processing: Use "Split In Batches" to process data in chunks
  • Parallel Execution: Use "Merge" node to run multiple branches in parallel
  • Cache Results: Store intermediate results to avoid re-processing
  • Optimize Code: Avoid nested loops, use efficient algorithms
  • Reduce API Calls: Batch API requests when possible
  • Use Faster Models: Consider gpt-3.5-turbo instead of gpt-4 for faster responses

🐌 Large PDF files take too long to process

Solution:
  • Process PDFs page by page using page ranges
  • Split large PDFs into smaller chunks
  • Use "Split In Batches" node for multi-page processing
  • Consider processing during off-peak hours
  • Implement async processing with webhooks for status updates

Connection and Network Issues

❌ Cannot connect to external APIs

Solution:
  1. Check internet connectivity
  2. Verify API endpoints are accessible
  3. Check firewall/proxy settings (for self-hosted n8n)
  4. Test API endpoints with curl or Postman
  5. Check if API requires specific IP whitelisting
  6. Verify SSL certificates are valid

❌ Self-hosted n8n not accessible

Solution:
  • Check if n8n process is running
  • Verify port 5678 (default) is open and not blocked
  • Check firewall rules
  • Verify N8N_HOST and N8N_PORT environment variables
  • Check Docker container status (if using Docker)
  • Review n8n logs for errors

Data Quality Issues

❌ Extracted data is incomplete or inaccurate

Solution:
  • Improve Prompts: Refine OpenAI prompts with more specific instructions
  • Use JSON Schema: Define expected output structure in OpenAI node
  • Post-Processing: Add validation and cleaning in Code nodes
  • Test with Samples: Test with various PDF formats to identify patterns
  • Manual Review: Review sample outputs and adjust extraction logic
  • Use Better Models: Try gpt-4 instead of gpt-3.5 for better accuracy

❌ Relationships are not correctly identified

Solution:
  • Enhance OpenAI prompt to specifically request relationship extraction
  • Add validation logic to check relationship integrity
  • Use fuzzy matching for name variations
  • Implement relationship validation in Code nodes
  • Add manual correction workflow for edge cases
