1. Webhook Node Configuration

Purpose

Receives PDF files via HTTP POST request. This is the entry point for your workflow.

Step-by-Step Configuration

1 Add Webhook Node
  • Press Space or click "+" button
  • Search for "Webhook"
  • Click to add to canvas
2 Configure HTTP Method
Field: HTTP Method
Value: POST
Location: Parameters tab → HTTP Method dropdown
3 Set Path
Field: Path
Value: pdf-upload
Location: Parameters tab → Path field

This creates the endpoint: https://your-n8n-instance.com/webhook/pdf-upload

4 Configure Response
Field: Response Mode
Value: Last Node
Location: Parameters tab → Response Mode dropdown

This means the response will come from the last node in your workflow.

5 Save and Get Webhook URL
  • Click "Save" button
  • Activate workflow (toggle "Active" switch)
  • Copy the Webhook URL displayed in the node
💡 Tip: The webhook URL is only available after saving and activating the workflow. Keep this URL secure as it can trigger your workflow.
📋 Example Configuration: 
HTTP Method: POST
Path: pdf-upload
Response Mode: Last Node
Response Code: 200

2. HTTP Request Node (LlamaParse API)

Purpose

Calls the LlamaParse API to extract text from the uploaded PDF file.

Step-by-Step Configuration

1 Add HTTP Request Node
  • Connect from Webhook node output
  • Search for "HTTP Request"
  • Add to workflow
2 Configure Method and URL
Field: Method
Value: POST
Field: URL
Value: https://api.llamaindex.ai/api/parsing/upload
3 Set Up Authentication
  • Click "Authentication" dropdown
  • Select "Generic Credential Type"
  • Click "Create New Credential"
  • Name: "LlamaParse API Key"
  • Add Header: 
    Name: Authorization
Value: Bearer YOUR_API_KEY
  • Click "Save"
4 Configure Request Body
Field: Body Content Type
Value: Binary Data
Field: Binary Data
Value: {{ $binary.data }}

This passes the PDF file from the webhook to LlamaParse.

5 Add Headers (if needed)
Field: Add Header
Name: Content-Type
Value: application/pdf
⚠️ Important: Make sure you have a valid LlamaParse API key. Sign up at llamaindex.ai to get your API key.
📋 Example Configuration: 
Method: POST
URL: https://api.llamaindex.ai/api/parsing/upload
Authentication: Bearer Token
Body: Binary Data from Webhook
Headers: Content-Type: application/pdf

3. Wait Node Configuration

Purpose

Waits for LlamaParse to finish processing the PDF before retrieving the results.

Step-by-Step Configuration

1 Add Wait Node
  • Connect after HTTP Request node
  • Search for "Wait"
  • Add to workflow
2 Configure Wait Type
Field: Resume
Value: After Time Interval
Field: Amount
Value: 5
Field: Unit
Value: Seconds

This waits 5 seconds before checking if parsing is complete. Adjust based on PDF size.

💡 Alternative: For more reliable polling, use a Loop node that checks the parsing status and waits until complete.

4. OpenAI Node Configuration

Purpose

Uses AI to extract organizational structure from the parsed PDF text.

Step-by-Step Configuration

1 Add OpenAI Node
  • Connect after getting parsed text from LlamaParse
  • Search for "OpenAI"
  • Select "OpenAI" node
2 Set Up Credentials
  • Click "Credential to connect with" dropdown
  • Click "Create New Credential"
  • Select "OpenAI API"
  • Enter your OpenAI API Key
  • Click "Save"
3 Configure Operation
Field: Operation
Value: Chat
Location: Parameters tab
4 Select Model
Field: Model
Value: gpt-4-turbo-preview or gpt-4o
Location: Parameters tab → Model dropdown

gpt-4-turbo provides better extraction accuracy for complex organizational charts.

5 Configure System Prompt
Field: Add Message → System Message
Value: 
You are an expert at analyzing organizational charts. Extract all staff members, their positions, departments, and reporting relationships. Return the data in a structured JSON format.
6 Configure User Prompt
Field: Add Message → User Message
Value: {{ $json.body.text }}

This passes the parsed text from LlamaParse to OpenAI.

7 Set Response Format
Field: Response Format
Value: JSON Object
Field: JSON Schema (optional)
Value: Define your expected JSON structure
8 Configure Temperature
Field: Temperature
Value: 0.2

Lower temperature (0.2) ensures more consistent, structured output.

📋 Example System Prompt: 
You are an expert at analyzing organizational charts. Extract the following information:
1. All employee names
2. Job titles/positions
3. Department names
4. Reporting relationships (who reports to whom)
5. Organizational hierarchy levels

Return the data as a JSON object with this structure:
{
  "staff": [
    {
      "name": "string",
      "title": "string",
      "department": "string",
      "reportsTo": "string",
      "level": number
    }
  ],
  "departments": [...],
  "reportingRelationships": [...]
}

5. Code Node (JavaScript) Configuration

Purpose

Processes and transforms data using custom JavaScript code. Used for parsing responses, building relationships, and generating JSON.

Step-by-Step Configuration

1 Add Code Node
  • Search for "Code"
  • Select "Code" node
  • Add to workflow
2 Select Language
Field: Language
Value: JavaScript
Location: Parameters tab
3 Write Your Code

In the code editor, write your JavaScript. Available variables:

  • $input - All input data
  • $json - JSON data from previous node
  • $binary - Binary data from previous node
  • $node - Data from specific node by name
📋 Example: Parse OpenAI Response 
// Get the OpenAI response
const openaiResponse = $json.message.content;

// Parse the JSON string
let orgData;
try {
  orgData = JSON.parse(openaiResponse);
} catch (error) {
  // If response is not valid JSON, try to extract it
  const jsonMatch = openaiResponse.match(/\{[\s\S]*\}/);
  if (jsonMatch) {
    orgData = JSON.parse(jsonMatch[0]);
  } else {
    throw new Error('No valid JSON found in response');
  }
}

// Return the parsed data
return {
  json: orgData
};
📋 Example: Build Relationship Graph 
// Get staff data from previous node
const staff = $json.staff || [];

// Build reporting relationships
const relationships = [];
const staffMap = new Map();

// Create staff map for quick lookup
staff.forEach(person => {
  staffMap.set(person.name, person);
});

// Build relationships
staff.forEach(person => {
  if (person.reportsTo) {
    const manager = staffMap.get(person.reportsTo);
    if (manager) {
      relationships.push({
        manager: {
          name: manager.name,
          title: manager.title,
          department: manager.department
        },
        subordinate: {
          name: person.name,
          title: person.title,
          department: person.department
        }
      });
    }
  }
});

return {
  json: {
    relationships: relationships,
    staff: staff
  }
};

6. Write Binary File Node Configuration

Purpose

Saves the generated JSON to a file on the file system.

Step-by-Step Configuration

1 Add Write Binary File Node
  • Search for "Write Binary File"
  • Add to end of workflow
2 Configure File Path
Field: File Name
Value: {{ $json.metadata.sourceFile.replace('.pdf', '_org_chart.json') }}
Location: Parameters tab

This creates a filename based on the original PDF name.

3 Set File Data
Field: Data
Value: {{ JSON.stringify($json, null, 2) }}

This converts the JSON object to a formatted string.

4 Set File Path
Field: File Path (if available)
Value: public/pdfs/California/state/cdt/json/

Make sure this directory exists and n8n has write permissions.

⚠️ Important: For self-hosted n8n, ensure the file system path is accessible. For cloud n8n, you may need to use a different storage solution (e.g., Google Drive, S3).
📋 Alternative: Using HTTP Request to Save

If Write Binary File isn't available, you can use an HTTP Request node to POST the JSON to your own API endpoint that saves the file.

7. If Node (Conditional Logic)

Purpose

Adds conditional logic to your workflow, allowing different paths based on conditions.

Step-by-Step Configuration

1 Add If Node
  • Search for "If"
  • Add where you need conditional logic
2 Configure Condition
Field: Condition
Value 1: {{ $json.staff.length }}
Operation: Larger
Value 2: 0

This checks if staff data was extracted (length > 0).

3 Connect Output Paths
  • True path: Connect to success handling
  • False path: Connect to error handling
📋 Example Conditions: 
// Check if parsing was successful
{{ $json.status }} === 'completed'

// Check if staff data exists
{{ $json.staff && $json.staff.length > 0 }}

// Check for errors
{{ $json.error === undefined }}

8. Error Trigger Node

Purpose

Catches errors from any node in the workflow and handles them gracefully.

Step-by-Step Configuration

1 Add Error Trigger
  • Search for "Error Trigger"
  • Add to workflow
2 Configure Error Handling
Field: Error Types
Value: All (or select specific error types)
3 Add Error Handling Actions

Connect nodes to handle errors:

  • Log error to file
  • Send notification email
  • Save error details to database

📋 Example: Log Error 
// In a Code node after Error Trigger
const error = $json.error;
const timestamp = new Date().toISOString();

return {
  json: {
    timestamp: timestamp,
    error: error.message,
    stack: error.stack,
    node: error.node,
    workflow: error.workflow
  }
};

Best Practices

✅ Naming Nodes

Always give your nodes descriptive names (e.g., "LlamaParse Upload", "Extract Org Structure", "Save JSON File"). This makes debugging easier.

✅ Testing Incrementally

Test each node individually before connecting to the next. Use "Execute Node" to test with sample data.

✅ Using Expressions

Click the "Expression" button (fx icon) next to any field to write JavaScript expressions that reference data from previous nodes.

⚠️ API Keys Security

Never hardcode API keys in nodes. Always use n8n's credential system to store sensitive information securely.

✅ Error Handling

Always add error handling nodes to catch and log errors. This helps with debugging and monitoring.
