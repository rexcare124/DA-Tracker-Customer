Overview

This plan outlines a workflow using n8n to process PDF organizational charts, extract staff information and reporting relationships, and generate structured JSON files. The workflow utilizes LlamaIndex (specifically LlamaParse, which is part of the LlamaIndex platform) for PDF parsing and text extraction. The plan is based on workflow templates from n8n creator Mark Shcherbakov (@lowcodingdev), particularly the "Parse PDF with LlamaParse" template.

Workflow Architecture

Phase 1: PDF Input and Parsing

Step 1.1: File Input

  • Node Type: Webhook or File System
  • Purpose: Accept PDF files (via webhook upload or file system monitoring)
  • Configuration:
    • Accept PDF files from specified directory: public/pdfs/California/state/cdt/
    • Support multiple file formats if needed
    • Validate file type and size

Step 1.2: PDF Text Extraction

  • Node Type: HTTP Request (LlamaParse API) or PDF Parser
  • Service: LlamaParse from LlamaIndex (as demonstrated in template 10634)
  • Purpose: Extract text content from PDF organizational charts
  • Configuration:
    • API endpoint: LlamaParse API
    • Parse mode: "text" or "markdown" for structured extraction
    • Handle multi-page documents
    • Preserve formatting and structure

Alternative Options:

  • PyPDF2/pdfplumber: If using Python Code node
  • Adobe PDF Services API: For enterprise-grade extraction
  • Tesseract OCR: If PDFs are image-based

Phase 2: Text Processing and Structure Recognition

Step 2.1: Text Cleaning and Normalization

  • Node Type: Code (JavaScript) or Function
  • Purpose: Clean extracted text, normalize whitespace, remove artifacts
  • Operations:
    • Remove page headers/footers
    • Normalize line breaks
    • Remove excessive whitespace
    • Handle special characters

Step 2.2: Organizational Structure Detection

  • Node Type: OpenAI or Code
  • Purpose: Identify organizational hierarchy patterns
  • AI Prompt Strategy: 
    Analyze the following organizational chart text and identify:
1. Top-level executives/directors
2. Department/division names
3. Manager-subordinate relationships
4. Reporting lines and hierarchy levels
5. Position titles and names

Extract this information in a structured format.

Step 2.3: Entity Extraction

  • Node Type: OpenAI (with structured output) or Code
  • Purpose: Extract specific entities (names, titles, departments)
  • Extraction Targets:
    • Employee names
    • Job titles/positions
    • Department/division names
    • Reporting relationships
    • Organizational levels

Phase 3: Relationship Mapping

Step 3.1: Relationship Identification

  • Node Type: Code (JavaScript) or Function
  • Purpose: Map reporting relationships from extracted data
  • Logic:
    • Identify manager-subordinate pairs
    • Map department hierarchies
    • Create relationship graph
    • Handle multiple reporting lines if present

Step 3.2: Hierarchy Construction

  • Node Type: Code (JavaScript)
  • Purpose: Build hierarchical tree structure
  • Algorithm:
    • Start from top-level positions
    • Recursively build down the hierarchy
    • Handle cross-department relationships
    • Validate structure integrity

Phase 4: JSON Generation

Step 4.1: Data Structure Assembly

  • Node Type: Code (JavaScript)
  • Purpose: Assemble extracted data into JSON structure

JSON Schema:

{
  "document": {
    "title": "CDT Organizational Chart",
    "source": "California Department of Technology",
    "extractionDate": "ISO 8601 date",
    "totalPages": number
  },
  "organizationalStructure": {
    "topLevel": {
      "position": "string",
      "name": "string",
      "department": "string"
    },
    "departments": [...],
    "reportingRelationships": [...]
  },
  "staff": [...]
}

Step 4.2: JSON File Generation

  • Node Type: Write Binary File or HTTP Request
  • Purpose: Save JSON to file system
  • Configuration:
    • Output directory: public/pdfs/California/state/cdt/json/
    • File naming: {original_filename}_org_chart.json
    • Format: Pretty-printed JSON with indentation
    • Encoding: UTF-8

Phase 5: Validation and Error Handling

Step 5.1: Data Validation

  • Node Type: Code (JavaScript) or If node
  • Purpose: Validate extracted data quality
  • Checks:
    • Required fields present
    • Relationship integrity
    • No orphaned positions
    • Valid JSON structure

Step 5.2: Error Handling

  • Node Type: Error Trigger or Try-Catch
  • Purpose: Handle processing failures gracefully
  • Actions:
    • Log errors with context
    • Retry failed operations
    • Generate error reports
    • Notify administrators

Detailed n8n Workflow Steps

Workflow 1: Basic PDF to JSON Conversion

1. Webhook (POST) - Receive PDF file
   ↓
2. HTTP Request - Call LlamaParse API
   - Method: POST
   - URL: https://api.llamaindex.ai/api/parsing/upload
   - Body: PDF file binary
   ↓
3. Wait - Poll for parsing completion
   ↓
4. HTTP Request - Get parsed text
   - Method: GET
   - URL: {parsing_result_url}
   ↓
5. Code (JavaScript) - Extract organizational data
   - Use regex/string parsing
   - Identify names, titles, relationships
   ↓
6. Code (JavaScript) - Build JSON structure
   ↓
7. Write Binary File - Save JSON
   - Path: public/pdfs/California/state/cdt/json/

Workflow 2: AI-Enhanced Extraction (Recommended)

1. Webhook (POST) - Receive PDF file
   ↓
2. HTTP Request - Call LlamaParse API
   ↓
3. Wait - Poll for parsing completion
   ↓
4. HTTP Request - Get parsed text
   ↓
5. OpenAI - Extract organizational structure
   - Model: gpt-4-turbo or gpt-4o
   - System Prompt: "You are an expert at analyzing organizational charts..."
   - User Prompt: "Extract all staff, positions, and reporting relationships..."
   - Response Format: JSON schema
   ↓
6. Code (JavaScript) - Parse OpenAI response
   ↓
7. Code (JavaScript) - Build relationship graph
   ↓
8. Code (JavaScript) - Generate final JSON
   ↓
9. Write Binary File - Save JSON

Workflow 3: Multi-Page Processing

1. Webhook (POST) - Receive PDF file
   ↓
2. HTTP Request - Call LlamaParse API (with page range)
   ↓
3. Split In Batches - Process each page separately
   ↓
4. For Each Page:
   - HTTP Request - Get page text
   - OpenAI - Extract page data
   - Code - Structure page data
   ↓
5. Code (JavaScript) - Merge page data
   ↓
6. Code (JavaScript) - Generate final JSON
   ↓
7. Write Binary File - Save JSON

Core Nodes:

  1. Webhook - Receive PDF uploads

  3. Code (JavaScript) - Custom data processing
  4. Write Binary File - Save JSON files

  6. Wait - Polling for async operations
  7. Split In Batches - Process multiple pages

External Services:

  1. LlamaParse API - PDF text extraction (part of LlamaIndex)
    • Sign up at: https://www.llamaindex.ai/

    • Model: gpt-4-turbo or gpt-4o for best results
    • Structured outputs for consistent JSON
  3. Alternative: Supabase - If storing intermediate data
    • Vector search for document retrieval
    • Database for relationship storage

Testing Strategy

Test Cases:

  1. Single-page chart - Simple hierarchy
  2. Multi-page chart - Complex structure across pages
  3. Image-based PDF - Requires OCR
  4. Various formats - Different chart layouts
  5. Edge cases:
    • Missing information
    • Ambiguous relationships
    • Cross-department reporting

Validation:

  • JSON schema validation
  • Relationship integrity checks
  • Completeness verification
  • Manual review of sample outputs

Optimization Recommendations

  1. Caching: Cache parsed PDF text to avoid re-parsing
  2. Batch Processing: Process multiple PDFs in parallel
  3. Incremental Updates: Only process changed pages
  4. Error Recovery: Resume from last successful page
  5. Performance: Use streaming for large files

Cost Considerations

LlamaParse:

  • Free tier available
  • Pay-per-page for production use
  • Estimate: $0.003-0.01 per page

OpenAI:

  • GPT-4 Turbo: ~$0.01-0.03 per 1K tokens
  • Estimate: $0.10-0.50 per organizational chart
  • Use gpt-3.5-turbo for cost savings if acceptable

n8n:

  • Self-hosted: Free (open source)
  • Cloud: Pay-per-execution or subscription

Security Considerations

  1. API Keys: Store securely in n8n credentials
  2. File Access: Restrict file system permissions
  3. Data Privacy: Handle PII (names, emails) appropriately
  4. Access Control: Secure webhook endpoints
  5. Audit Logging: Track all processing activities

Maintenance and Updates

  1. Monitor API Changes: LlamaParse and OpenAI may update APIs
  2. Refine Prompts: Continuously improve extraction accuracy
  3. Handle New Formats: Adapt to different chart layouts
  4. Performance Monitoring: Track processing times and costs
  5. Error Analysis: Review failed extractions for improvements
